## Summary

Formal definitions of all official BSM metrics.

## Specification

### Accuracy

```
ACC = correct_predictions / total_predictions
```

### Recall@k

```
R@k = |{q ∈ Q : correct_item(q) ∈ top_k(q)}| / |Q|
```

### Latency

L₅₀ = median retrieval time in microseconds.
L₉₉ = 99th percentile retrieval time in microseconds.

### Memory Footprint

V = sizeof(keys) + sizeof(values) + sizeof(metadata) + sizeof(indices)

All in bytes.

### Knowledge Density

```
KD = (H_prior - H_posterior) / V

H_prior = log₂(n_classes)
H_posterior = -Σ p(c|M) · log₂ p(c|M)
```

Where p(c|M) is the empirical distribution of predictions given memory M.

### Participation Ratio

```
PR = (Σ λ_i)² / Σ λ_i²

λ = eigenvalues of cov({S_i})
```

### Consolidation Gain

```
CG = ACC_after_sleep / ACC_before_sleep
```

### Forgetting Rate

```
FR = entries_forgotten / total_hours_runtime
```

## Rationale

Standard definitions ensure that metric values are comparable across
implementations, papers, and releases.

## Compatibility

All metrics are dimensionless except Latency, Memory, and Forgetting Rate.
