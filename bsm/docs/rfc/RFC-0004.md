## Summary

Defines the official BSM-Bench benchmark suite.

## Specification

### Datasets

| Dataset | Source | N_train | N_test | Domain |
|---------|--------|---------|--------|--------|
| tinystories | TinyStories | 2000 | 500 | Language |
| wikitext | WikiText-2 | 2000 | 500 | Language |
| python_code | Public repos | 2000 | 500 | Code |
| dna | Human genome | 2000 | 500 | DNA |
| random | Uniform random | 2000 | 500 | Null baseline |

### Metrics Collected

- Accuracy
- Recall@1, Recall@5
- Latency p50, p99 (µs)
- Memory footprint (bytes)
- Knowledge Density (bits/byte)
- Participation Ratio
- Build time (ms)

### Procedure

1. For each dataset:
   a. Train/encode 2000 entries
   b. Measure build time
   c. Run 500 queries, measure latency
   d. Report all metrics
2. Repeat 3 times, report mean.
3. Fixed seed = 42.

### Output

- `report.json`: machine-readable
- `report.md`: human-readable
- `report.html`: interactive (optional)

## Rationale

Standardised benchmarks ensure reproducible comparisons across versions.
Five datasets cover the domains where BSM's geometry varies most.

## Compatibility

New datasets can be added without changing the benchmark runner.
