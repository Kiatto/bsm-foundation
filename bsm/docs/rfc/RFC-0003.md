## Summary

Defines the serialization format for Memory Store and Router.

## Specification

### Memory Store

**File extension:** `.bsm-store.npz`

**Contents:**

| Key | Type | Description |
|-----|------|-------------|
| `state_dim` | int | State dimension D |
| `n_uint64` | int | Number of uint64 per state |
| `keys` | (N, n_uint64) uint64 | Packed binary states |
| `n_entries` | int | Number of entries |

Values are stored in a companion file: `.bsm-store.vals.jsonl`
(one JSON line per entry, each line `{"v": <payload>, "meta": <dict>}`).

### Router

**File extension:** `.bsm-router.npz`

**Contents:**

| Key | Type | Description |
|-----|------|-------------|
| `state_dim` | int | State dimension D |
| `proto_{name}` | (D,) int8 | One array per route |

### Versioning

A format version field (`format_version: int`) SHOULD be included
in all files.  Current version: 1.

## Rationale

NumPy's `.npz` format is ubiquitous, zero-dependency, and supports
compression.  Separating values into JSONL allows arbitrary payload
types without serialization coupling.

## Compatibility

Loaders MUST support format_version 1.  Unknown keys MUST be ignored.
