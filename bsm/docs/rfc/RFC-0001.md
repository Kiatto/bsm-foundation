## Summary

Defines the protocol for storing and retrieving entries in BSM memory.

## Specification

### Operations

1. `observe(state, payload, value=1.0)`
   - `state`: (D,) int8 array in {-1, +1}
   - `payload`: any JSON-serializable object
   - `value`: float in [0, 1], initial confidence
   - Returns: None

2. `recall(state, k=5)`
   - `state`: (D,) int8 array
   - `k`: positive integer
   - Returns: list of (payload, hamming_distance, metadata)

3. `predict(state, decoder_fn=None)`
   - Returns: most common payload among nearest neighbours

### Storage

Entries are stored as (state, payload, metadata) tuples.
State is the key; Hamming distance is the comparison function.

## Rationale

A minimal protocol that covers both episodic storage (observe)
and content-addressable retrieval (recall).  No query language,
no filters, no projections — just nearest-neighbour in Hamming space.

## Compatibility

Backward compatible with BSM Memory Engine v1.0.
