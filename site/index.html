A memory that tells you in advance how well it will answer.

ABM is a deterministic memory runtime for compiled knowledge. Unlike vector memories, it predicts — before you run a single query — how accurately it can answer under a fixed memory budget. Then you can hold it to that number.

GitHub 90-second live demo

INSTALL · 30 SECONDS

pip install abm-runtime
abm demo

USE · 5 MINUTES

from abm import Memory, contract

mem = Memory(dim=8192)
mem.store("payment_service", "requires", "auth_service")
mem.store("auth_service", "writes_to", "session_store")

mem.query("payment_service", "requires")
# ("auth_service", 0.99)   calibrated: 0.5 = random

mem.chain("payment_service", ["requires", "writes_to"])
# ("session_store", 0.98)  multi-hop, no index, no search

print(contract(mem, grounding=0.93))

THE MEMORY CONTRACT

MEMORY CONTRACT
  Capacity        <= 944 facts (D=8192, codebook=74)
  Expected accuracy >= 96% at current load (180 facts)
  Grounding       >= 93% (projected end-to-end 89%)
  Confidence      calibrated (0.5 = chance), margin 3.1σ
  Max depth (p50) =  7 hops
  Pressure        =  0.19

Every line is computed from closed-form capacity laws — zero fitted parameters. If the contract says 96% and you measure 60%, that's a bug in ABM, not bad luck. The contract is the product.

FAQ

Is this a vector database?

No. There are no embeddings and no similarity search. Facts are exact (subject, relation, object) triples bound into a single binary trace by XOR algebra. It complements vector search; it doesn't replace it.

Do I need to understand the math?

No. Five concepts: Memory, store, query, chain, contract. The theory exists so the contract can be computed — you only read its output.

Where do the facts come from?

From you, or from an LLM reading your documents. That extraction step is honestly the hard part of any symbolic pipeline — which is exactly why the contract accepts your extractor's audited precision and projects the end-to-end number.

What's the catch?

Knowledge must be compiled into triples before it's useful, and entities are exact names — normalize at ingestion. If your data can't be expressed as discrete facts, ABM is the wrong tool and we'll say so.

How is the accuracy prediction possible?

The memory is a mathematical object with known statistics; its capacity and error rates follow closed-form laws that were published, pre-registered and falsification-tested. The repo contains every experiment, including the failed ones.
